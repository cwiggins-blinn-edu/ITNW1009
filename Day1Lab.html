1. Accessing the AWS Academy Course

Enroll in and log into your official AWS Academy Learner Lab course space.

Before You Start

You will need your college student email and the course join code provided by your instructor (posted in the class LMS or announced in the syllabus). Do not use a personal email address — AWS Academy access is tied to your official student account.

  1. 1

    Check your email for an invitation

    Look for a course invitation email from awsacademy.instructure.com. If you don't see it, check your spam/junk folder or ask your instructor to resend the invite.

  2. 2

    Create or sign in to your AWS Academy account

    Navigate to awsacademy.instructure.com. First-time users will create an account using their student email; returning users simply log in with their existing credentials.

  3. 3

    Enter your course join code

    If prompted, enter the join code your instructor shared for ITNW 1009. This links your account to the correct class roster and unlocks your Learner Lab access.

  4. 4

    Confirm enrollment on the course dashboard

    Once logged in, confirm that ITNW 1009 appears on your Dashboard or Courses list. Click into the course and locate the Modules page — this is where weekly labs, including today's, will be posted.

Stuck getting in? If your invitation link has expired or your join code doesn't work, email your instructor with your full name and student email so they can re-send access — don't try creating a second account.

3. Navigating the AWS Cloud Environment & Inspecting Utility Dashboards

Start your sandbox, tour the Management Console, and locate billing & security essentials.

Lab Overview & Objectives

This lab introduces you to the live AWS Management Console using your sandboxed AWS Academy student environment. You will learn to safely start your lab instance, access the cloud environment, find core services, read regional service distributions, and view real-time sandbox financial tracking configurations.

01

Start the AWS Academy student learning lab sandbox environment safely.

02

Navigate the global AWS Management Console layout using the service search features.

03

Identify deployment region locations and locate the account's pre-provisioned security frameworks.

04

Access the billing dashboard interface to check spending indicators and resource boundaries.

Critical Lab Environment Constraints

Before clicking any buttons, review these rules to prevent system errors.

Critical Restriction

Do not create custom security or billing structures

You are working inside a highly restricted student environment. Do NOT attempt to create custom IAM Users, custom IAM Groups, custom IAM Policies, or custom Root Billing structures. Doing so will violate your lab permissions and immediately throw an Access Denied error message.

Required Security Profile

Whenever you launch compute resources, databases, or storage components in future labs, you must always select and attach the pre-provisioned security role named LabRole. Do not create new roles.

Automated Checkpoints

This lab utilizes an integrated AWS instruction panel check system. You can verify your configuration tasks in real time by clicking the explicit check buttons located in your student manual interface.

Step-by-Step Lab Execution Instructions

Task 1: Starting the Sandbox Lab Environment

  1. Open your web browser and navigate to your official AWS Academy Learner Lab portal dashboard page.
  2. Locate the floating command control menu at the top of your course screen. Find the text option labeled Start Lab accompanied by a small status circle icon.
  3. Click Start Lab once. The small status indicator light changes to an amber color, indicating that the backend AWS provisioning engines are building your isolated student lab account environment.
  4. Wait patiently for the status circle icon to turn solid green. Once it turns green, look for the text option labeled AWS next to a green dot indicator.
  5. Click the AWS link option. A new browser tab will launch automatically, logging you directly into the live, fully functional AWS Management Console control homepage.

Task 2: Navigating the Global AWS Management Console Layout

  1. Look at the dark navigation bar running across the very top of your browser tab page. This is your primary control anchor panel.
  2. Locate the central text input field box labeled Search (keyboard shortcut alternative: forward slash key /).
  3. Type the keyword text IAM directly into the search bar field.
  4. Review the filtered dropdown results. Under the primary category header labeled Services, look for the option labeled IAM (Identity and Access Management — Securely Manage Access to AWS Services and Resources).
  5. Click the IAM option to load the central Identity control dashboard page.
  6. Look closely at the IAM page interface display. Note that the system alerts you with a status line indicating that IAM is a Global Service. This means Identity configurations do not require regional selection filters; they apply automatically across the entire global AWS infrastructure ecosystem.

Task 3: Verifying the Pre-Provisioned Lab Security Profile

  1. While remaining on the main IAM control dashboard page, look at the left-hand vertical navigation hierarchy panel column.
  2. Locate and click on the menu folder option explicitly labeled Roles.
  3. Locate the list search filter text field box situated directly above the main table display window.
  4. Type the string search phrase LabRole into the filter field box and press your keyboard Enter key.
  5. Verify that a pre-configured role entry named LabRole appears clearly within the data rows. Click on the text name link for LabRole.
  6. Inspect the summary layout configuration details. Notice that this specialized security profile is automatically managed by the AWS Academy team. It contains the broad administrative cloud permissions required to complete your upcoming weekly course lab builds, saving you from having to build or manage complex security access policies yourself.

Task 4: Inspecting Regional Settings and the Account Billing Dashboard

  1. Shift your focus back up to the primary dark navigation control anchor bar located at the very top right of your browser page display window.
  2. Find the geographic location text selector located directly to the right of your student account name details (this selector often displays default region names such as N. Virginia or Oregon).
  3. Click the regional dropdown button to view a list of available global data collection deployments across the world. Ensure your selector remains set to the course-assigned region (typically us-east-1 N. Virginia or us-west-2 Oregon as directed by your screen instructions).
  4. Click directly on your personal user account identity name icon, located on the far right edge of the top dark navigation bar area.
  5. From the dropdown context menu that appears, locate and click on the option explicitly labeled Billing Dashboard.
  6. Review the central ledger overview page display screen. Locate the analytics display sections tracking Summary costs and AWS Free Tier usage consumption limits.
  7. Observe that your student lab environment features a fixed operational ceiling budget limit (typically a $100.00 credit allocation). This dashboard is where you will track your real-time cloud resource consumption expenses over the next few weeks to ensure you don't run out of lab funds before the end of the term.

Common Lab Troubleshooting Guidance

Observed Error Symptoms Identified Core Root Cause Step-by-Step Remediation Action
Screen displays an explicit Access Denied error banner box when clicking on a service component. Attempting to create unauthorized configurations (such as a custom IAM user or a manual billing alert parameter). Immediately cease the action. Navigate backward to the main dashboard menu using the top search bar and stick strictly to the pre-allocated lab tasks.
The AWS icon link dot remains dark or unclickable, preventing console initialization. The student backend lab sandbox environment has not finished spinning up yet. Return to the portal manual window, check if the status indicator is still amber, wait 60 seconds, and refresh your web page.
Missing or empty resource tables inside a specific service interface view. The region selection dropdown filter has inadvertently swapped to an alternate geographic zone. Click the location selector dropdown menu located at the top-right of the screen and switch back to your designated primary region zone.
